# Юзербот-автокомментатор (личный аккаунт)

Пишет одну из ваших фраз (случайно выбранную) в комментарии под постами вашего
канала — от имени вашего реального Telegram-аккаунта, не бота.

## ⚠️ Что важно понимать

- Это работает через ваш **личный аккаунт**, а не через Bot API. Для входа нужны номер телефона и код подтверждения (как в обычном Telegram-клиенте).
- Автоматизация личного аккаунта формально не входит в officially поддерживаемые Telegram сценарии использования. Для простого действия (один ответ на пост) риск невысокий, но исключить его нельзя — будьте готовы, что при подозрительной активности Telegram может временно ограничить аккаунт.
- **Строка сессии (SESSION_STRING) — это полный доступ к вашему аккаунту.** Никому её не показывайте, не публикуйте, не коммитьте в открытый git-репозиторий.

## Шаг 1. Получите API-ключи

1. Зайдите на https://my.telegram.org → **API Development Tools**.
2. Создайте приложение (любое название), получите `api_id` и `api_hash`.

## Шаг 2. Сгенерируйте сессию (локально, не на Railway!)

```bash
pip install telethon
python generate_session.py
```

Введите `api_id`, `api_hash`, номер телефона, код из Telegram (и пароль 2FA, если он у вас включён). Скрипт выведет длинную строку — это `SESSION_STRING`. Сохраните её.

## Шаг 3. Деплой на Railway

1. Залейте содержимое этой папки в свой GitHub-репозиторий.
2. Railway: **New Project → Deploy from GitHub repo**.
3. Во вкладке **Variables** добавьте:
   - `API_ID` — из шага 1
   - `API_HASH` — из шага 1
   - `SESSION_STRING` — из шага 2
4. Railway подхватит `Procfile` и запустит `python userbot.py`.

### Про хранение данных (группы и фразы)

Список подключённых групп хранится в файле `allowed_groups.json`, а список фраз — в `phrases.json`, рядом с кодом. На бесплатном плане Railway файловая система не сохраняется между деплоями — при каждом `git push` эти списки обнулятся, и настройку нужно будет повторить.
Чтобы это не терять — подключите **Volume** в Railway (Settings → Volumes) и укажите в `groups_store.py` и `phrases_store.py` пути вида `/data/allowed_groups.json` и `/data/phrases.json` вместо локальных файлов.

## Как пользоваться

1. Добавьте свой аккаунт (тот, под которым выполнен вход) в группу обсуждений вашего канала — как обычно добавляют участника.
2. Сделайте группу публичной, чтобы у неё был юзернейм (Settings группы → Group Type → Public).
3. В любом чате (например, в «Избранном» / Saved Messages) напишите от своего имени:
   ```
   /addgroup @username_группы
   ```
   Бот отредактирует ваше же сообщение, подтвердив подключение.
4. Готово — теперь при каждом новом посте канала в этой группе юзербот автоматически ответит одной из включённых фраз.

## Все команды (пишутся вами, от своего аккаунта, начинаются со слэша `/`)

Полный список выводится командой **`/help`**.

**Группы**
- `/addgroup @username` — подключить группу обсуждений
- `/removegroup @username` — отключить группу
- `/mygroups` — список подключённых групп

**Фразы для автокомментария**
- `/phrases` — показать все 5 фраз с их статусом (✅ включена / ⛔ выключена)
- `/setphrase <номер> <текст>` — изменить текст фразы по номеру (например: `/setphrase 2 🔥огонь`)
- `/enable <номер>` — включить фразу по номеру
- `/disable <номер>` — выключить фразу по номеру (можно оставить включённой только одну)

При каждом автопосте бот случайно выбирает одну из **включённых** фраз и отвечает ей. По умолчанию заданы 5 фраз: `1`, `🔥`, `👍`, `😁`, `+` — все включены; их текст и статус можно менять командами выше.

**Прочее**
- `/status` — проверить, что юзербот запущен, сколько групп подключено и сколько фраз включено
- `/help` — показать список всех команд

## Как это отличает "автоматический" пост канала от обычной пересылки

Когда пост из канала попадает в привязанную группу обсуждений автоматически, в группе он выглядит так, будто его "отправил" сам канал (а не пользователь). Юзербот проверяет именно это: и что сообщение переслано из канала, и что автор сообщения в группе — сам канал, а не человек, вручную переславший пост. Так бот не реагирует на случайные ручные репосты канала от других участников группы.
